# 04. OID-PPO 모델 — 신경망이 어떻게 생겼나 (4장)

## 큰 그림: 두 개의 눈

- **그래프 눈**: "무엇이 무엇과 관계있는지"를 봄. 가구 수가 4개든 8개든 상관
  없음.
- **지도 눈(CNN)**: 위에서 내려다본 격자 지도로 "어디가 좁고 어디가 비었는지"를
  봄.

역할 분담이 핵심입니다. 그래프는 좁은 틈까지 알 필요가 없고, 지도는 누가
누구의 짝인지 알 필요가 없습니다. 각자 잘하는 것만 하고, 아래의 FiLM으로
연결됩니다.

## 타입 그래프 (정의 6)

- **노드**: 가구 하나마다 1개 (이미 놓임/지금 놓는 중/대기 중 모두), 벽마다
  1개, 문마다 1개.
- **노드 특징**: 종류, 크기, 상태(놓임/현재/대기), 순서, 자세, 구조 기하.
- **엣지(선)**: 상대 위치 정보 + 타입 라벨 (공간 이웃 / 기능 짝 / 벽 / 문 /
  구조).
- **자기 자신으로의 셀프 루프**도 있어서 이웃이 아예 없는 노드가 생기지 않음.

대기 중인 가구도 노드로 넣는 이유: "아직 안 놓은 협탁이 침대 옆자리를 필요로
할 것"을 미리 알고 침대를 놓게 하기 위해서입니다.

## 타입 어텐션 — 수식 풀이

어텐션(attention)은 "각 노드가 이웃들에게 얼마나 주목할지"를 계산하는
방법입니다. 수식이 여러 줄이지만 구조는 표준 질의-키-값(query-key-value)
어텐션에 "타입 정보"를 더한 것입니다.

말로 풀면, 노드 i가 이웃 j를 볼 때:

1. **질의 q_i**: "나는 이런 정보가 필요해" (노드 i의 상태를 행렬 W_Q로 변환)
2. **키 k_ij**: "나는 이런 정보야" — 여기에 **엣지의 기하 정보**(r_ij: 상대
   위치 등)와 **엣지 타입 임베딩**(e_τ: "우리는 짝 관계" 같은 라벨의 학습된
   벡터)이 더해집니다.
3. **점수 e_ij**: 질의와 키의 내적(유사도) ÷ √(차원) + 타입별 보정치 b_τ.
   내적을 차원의 제곱근으로 나누는 건 숫자가 너무 커지지 않게 하는 표준
   기법입니다.
4. **가중치 α_ij**: 점수들을 softmax(지수화 후 합이 1이 되게 정규화)로 바꿈 —
   "이웃들에 대한 주목 배분율".
5. **메시지 m_i**: 이웃들의 값 v_ij를 가중 평균.
6. 여러 개의 머리(head)가 이걸 병렬로 하고(∥는 이어붙이기), 잔차 연결이 있는
   공유 신경망(ResFFN)으로 노드 상태를 갱신.

**타입이 들어가는 곳이 세 군데**(키, 값, 점수 보정)라서, "짝 엣지"와 "벽
엣지"는 완전히 다르게 처리됩니다.

마지막에 **그래프 요약 벡터 h^G**를 만듭니다: 학습된 질의 벡터로 전체 노드를
가중 평균(풀링)한 것에, **지금 놓는 가구의 노드 상태를 따로 이어붙임**. "방
전체 분위기 + 지금 내 손에 든 가구"를 한 벡터로 요약한 셈입니다.

**명제 4 (순서 무관성)**: 가구 목록의 저장 순서를 뒤섞어도 (1번 가구를 3번으로
바꿔 불러도) 정책의 출력이 정확히 같습니다. 어텐션 합과 풀링이 순서를 안 타기
때문입니다. 배치 문제에서 가구 목록의 순번은 아무 의미가 없어야 하므로, 이건
꼭 필요한 성질입니다.

## FiLM: 그래프가 지도를 조절한다 — 수식 풀이

지도 눈의 입력 텐서는 행동 격자와 같은 48×48 좌표계를 쓰고, 여러 장의 지도
채널(방 모양, 벽, 점유, 문, 파트너 위치, 여유 공간, 회전별 가능/선호 지도)로
구성됩니다. 이것들은 **관측일 뿐**, 행동을 사후 수정하는 규칙이 아닙니다.

FiLM(Feature-wise Linear Modulation) 수식:

```
(ζ, β) = W×h^G + b          ← 그래프 요약에서 채널별 조절값 계산
Ĥ = (1+ζ)⊙GN(H) + β        ← 각 채널을 ζ배 늘리고 β만큼 이동
H_next = H + R(Ĥ)           ← 잔차 CNN 블록
```

- GN은 그룹 정규화(숫자 스케일을 고르게 맞추는 표준 장치), ⊙는 채널별 곱.
- **비유**: 그래프 요약이 리모컨이 되어 CNN의 각 채널의 "볼륨(ζ)과
  밝기(β)"를 돌립니다. "지금 협탁 차례야" → 침대 옆을 강조하는 채널의 볼륨을
  올림.

마지막 로짓(logit, 확률화 전 점수)은 CNN 출력에 **선호 지도들**(짝/벽/동선/
문목 + 학습된 항)을 더한 것입니다. 각 지도에는 그래프 요약에서 계산된
**게이트**(g_P, g_W, …)가 붙어서, 상황마다 어떤 선호를 얼마나 반영할지 정책
스스로 정합니다. σ_t(현재 로짓들의 표준편차)를 곱해 스케일을 맞춥니다.
중요: 이 지도들은 **순위만 바꾸고**, 가능한 행동의 집합은 절대 안 바꿉니다.

## 마스크드 정책 — 수식 (10) 풀이

액터의 출력은 4×48×48 크기의 점수 상자(로짓 텐서)입니다 — 후보 자세마다 점수
하나.

샘플링 전 안정화:
1. 허용된 자리들(지지집합 V_t)의 점수만 모아 평균 ν와 표준편차 ς를 계산
2. `(점수 − 평균) ÷ (온도 × 표준편차)`로 표준화 — 온도(θ_tr)가 높으면 고른
   탐색, 낮으면 확신 있는 선택. ε_z는 표준편차가 0일 때 0으로 나누는 것을
   막는 아주 작은 수.

수식 (10) — 정책:

```
π(a|s) = exp(표준화 점수) ÷ (허용된 자리들의 exp 합)   … 허용된 자리
π(a|s) = 0                                             … 금지된 자리
```

즉 **금지 자리는 확률 0으로 못박고, 허용 자리끼리만 확률을 나눠 갖는
softmax**입니다. 마스크가 분포의 일부라는 말은 이 뜻입니다 — 뽑고 나서
고치는 게 아니라, 애초에 뽑힐 수 없음.

- 학습 중에는 이 분포에서 샘플링, 평가 때는 마스크된 원시 로짓의 argmax(최고
  점수 하나)로 결정적으로 선택.
- 버퍼에는 정수 행동과 그 로그 확률을 저장하고, 재생(replay) 때 같은 변환을
  재구성합니다.

**보조정리 2**: 허용 자리가 있고 점수가 유한하면 위 식은 진짜 확률분포이고
(합=1, 금지 자리 확률 0, 허용 자리 확률 양수), 뽑힌 번호의 dec 좌표가 그대로
저장됩니다.

**따름정리 1 (PPO 비율의 안전성)**: 마스크가 상태만의 함수(파라미터와
무관)라서, 예전 정책과 지금 정책의 지지집합이 같습니다. 따라서 뽑았던 행동의
"지금 확률 ÷ 예전 확률"이 항상 잘 정의되고(0으로 나누기 없음) 유한합니다.
PPO는 이 비율로 학습하므로, 이것이 학습이 수학적으로 성립하기 위한 조건입니다.

## 크리틱, GSAE, PPO — 수식 풀이

**크리틱 V_φ(s)**: "이 상태에서 앞으로 받을 점수가 얼마일까"를 예측하는
심판. 그래프 요약 + 지도 풀링에서 예측하며, 액터와 인코더를 공유하되 머리는
분리.

**GSAE (정의 7)**: 표준 GAE를 3장의 셰이핑 보상에 적용한 것.

```
δ_t = r_t + γ×V(다음 상태) − V(지금 상태)   ← "예상보다 얼마나 좋았나" (TD 오차)
A_t = δ_t + (γλ)δ_{t+1} + (γλ)²δ_{t+2} + …   ← 미래의 놀라움을 λ로 할인해 합산
```

λ(0~1)는 "먼 미래의 놀라움을 얼마나 믿을지"입니다. A_t(어드밴티지)가
양수이면 "그 행동은 평소보다 좋았다"는 뜻이라 확률을 올리고, 음수면 내립니다.

**PPO의 클립 목적함수**:

```
비율 ρ = 지금 정책의 확률 ÷ 뽑을 당시 정책의 확률
목적  = min( ρ×A, clip(ρ, 1−ε, 1+ε)×A )
```

**비유**: 좋았던 행동(A>0)의 확률을 올리되, 비율이 1+ε을 넘는 순간부터는
보상이 더 안 늘게 잘라서(clip) **한 번의 업데이트로 정책이 폭주하는 것을
막는** 안전벨트입니다. 여기에 크리틱의 예측 오차 줄이기(value regression)와,
허용 자리 위에서의 엔트로피 보너스(한 자리만 고집하지 않게)를 더해 학습합니다.

**효율 포인트**: 후보 9,216개의 점수가 **신경망 한 번의 순전파**로 전부
나옵니다. 탐색처럼 후보마다 목적함수를 호출하지 않습니다.

**남는 것 (Remark 1)**: 증명한 것은 상태·대칭성·지지집합·실행·보상의
"계약"이 맞다는 것까지입니다. PPO가 전역 최적에 수렴한다는 증명이 아니고,
기능 점수가 인간의 미적 취향 모델인 것도 아닙니다.
