# 02. 문제 정의 — 가구 배치를 "게임 규칙"으로 만들기 (2장)

이 장은 가구 배치라는 문제를 수학적 게임(MDP)으로 바꾸는 부분입니다. 논문의
수식을 하나씩 말로 풀겠습니다.

## MDP란? — 수식 풀이

논문: `M = ⟨S, A, P, R, γ, T_max⟩`

턴제 보드게임 한 판을 상상하세요. 여섯 개의 재료가 게임을 정의합니다.

- **S (상태 집합)**: 게임판이 보일 수 있는 모든 모습. 여기서는 "방 + 지금까지
  놓인 가구들 + 다음에 놓을 가구".
- **A (행동 집합)**: 둘 수 있는 모든 수. 여기서는 9,216개의 자리 번호.
- **P (전이)**: 수를 두면 게임판이 어떻게 바뀌는지의 규칙.
- **R (보상)**: 수를 둔 뒤에 받는 점수.
- **γ (감가율, 0<γ≤1)**: 나중에 받을 점수를 얼마나 할인해서 볼지. γ=1이면
  "지금 1점 = 나중 1점".
- **T_max (최대 턴 수)**: 가구가 N개면 최대 N턴.

한 판(에피소드)이 시작될 때 방, 가구 목록, 짝 관계, 놓는 순서가 정해지고,
매 턴 가구를 하나씩 놓습니다. 다 놓으면 성공 종료, 다음 가구를 놓을 자리가
아예 없으면 실패 종료입니다.

## 방, 벽, 문 (정의 1)

- **방 E**: 직사각형 여러 개를 이어 붙인 도형(ㄱ자 방도 가능). 수학적으로
  "축에 평행한 직사각형들의 합집합"입니다.
- **경계 상자 B = [x⁻,x⁺]×[y⁻,y⁺]**: 방 전체를 딱 감싸는 가장 작은 직사각형.
  x⁻는 방에서 가장 왼쪽 좌표, x⁺는 가장 오른쪽 좌표. y도 마찬가지.
- **문**: 문마다 "문이 열리면서 쓸고 지나가는 부채꼴 영역 S_d"가 정해져 있고,
  그 위에는 가구를 못 놓습니다.

실험에 쓴 방은 정사각형·직사각형·L자형이고, 문은 북쪽 벽 왼쪽에 있습니다.
정의 자체는 다른 문 위치도 허용하지만, **실험 증거는 이 종류의 방에
한정된다**고 논문이 스스로 선을 긋습니다.

## 가구와 짝 관계 (정의 2)

가구 하나마다: 종류(침대/책상/…), 가로 폭, 세로 깊이, "앞면"이 어느 쪽인지,
그리고 "벽에 붙는 종류인지 / 사람 접근 공간이 필요한 종류인지"가 정해져
있습니다.

- **회전 k ∈ {0,1,2,3}**: 반시계 방향으로 90°×k 돌리기. 즉 0°, 90°, 180°,
  270° 네 가지만.
- **자세(pose) g = (x, y, k)**: 중심 좌표 + 회전. 자세가 정해지면 가구가
  차지하는 바닥 직사각형 **P_f(g)** (풋프린트)와 앞면 방향 벡터 **n_f(g)**가
  자동으로 정해집니다.
- **짝 관계 e = (p, c, τ, d*, σ)**: 부모 p(침대), 자식 c(협탁), 관계 종류
  τ(side=옆에 / face=마주 보게), 바람직한 간격 d*, 그리고 간격을 얼마나
  엄격하게 볼지 정하는 σ.
- **놓는 순서 ϖ**: 부모가 자식보다 먼저 오도록 정해진 순서. 침대를 놓기 전에
  협탁부터 놓는 일이 없게 합니다.

## 상태 (정의 3) — 수식 풀이

논문: `s_t = (I, L_t, t, z_t)`

t번째 턴의 상태는 네 가지 묶음입니다.

1. **I**: 이 판의 설계도 전체 (방, 가구 목록, 짝 관계, 순서)
2. **L_t**: 지금까지 실제로 놓인 가구들과 그 자세 목록
3. **t**: 몇 번째 턴인지
4. **z_t**: 진행 중(active) / 완료(complete) / 실패(failed)

중요한 점: 신경망이 보는 그래프와 지도는 이 상태에서 **계산해 낸 관측**이지,
상태 그 자체가 아닙니다.

## 하드 유효(hard-valid) 자세 (정의 4) — 수식 풀이

어떤 자세가 "물리적으로 가능"하려면 세 조건을 모두 만족해야 합니다.

1. 풋프린트가 방 안에 완전히 들어감 (`P ⊆ E`)
2. 어떤 문의 스윙 영역과도 겹치는 **면적이 0** (`μ₂(P ∩ S_d) = 0`)
3. 이미 놓인 어떤 가구와도 겹치는 **면적이 0**

여기서 μ₂는 그냥 "넓이"입니다. "면적이 0"이라고 쓴 이유: **모서리가 서로 딱
닿는 것은 허용**하고(넓이 0), 실제로 파고드는 것(넓이 양수)만 금지하기
위해서입니다. 벽에 딱 붙이는 배치가 가능해야 하니까요.

주의: 하드 유효는 "놓을 수 있다"는 뜻일 뿐, "좋은 자리"라는 뜻이 아닙니다.
문 앞을 안 막고 안 겹쳐도, 접근 불가능한 구석일 수 있습니다.

## 행동 격자 — 9,216개 번호의 정체 (수식 (1) 풀이)

**핵심 아이디어**: 가구 크기를 반영한 격자를 만든다.

회전 k를 정하면 가구의 절반 폭 u_k, 절반 깊이 v_k가 정해집니다. 그러면 중심을
둘 수 있는 x 범위는 `[x⁻+u_k, x⁺−u_k]`입니다 — 이보다 바깥이면 가구가 방을
삐져나가니까요. 이 범위 위에 **양 끝을 포함해 균일한 지점 48개(G=48)** 를
찍고, c번째 지점의 x좌표를 만듭니다:

```
x = (왼쪽 한계) + (c / 47) × (범위 길이)
```

- c=0이면 왼쪽 벽에 딱 붙는 위치, c=47이면 오른쪽 벽에 딱 붙는 위치.
- 즉 "벽에 딱 붙이기"가 격자 안에 항상 포함됩니다. 이게 풋프린트-인지
  (footprint-aware) 격자의 요점입니다.
- y도 마찬가지 (row 0이 북쪽).

**번호 매기기 (enc)**:

```
enc(k, r, c) = ((k×48) + r) × 48 + c
```

이건 4층짜리 주차타워의 자리 번호와 같습니다: k층(회전), r행, c열의 자리에
일련번호 하나를 붙인 것. 번호 범위는 0 ~ 4×48²−1 = 0 ~ 9,215.

- **idx**: 번호 → (k, r, c)로 되돌리기 (몇 층, 몇 행, 몇 열인지)
- **dec**: 번호 → 실제 좌표 (x, y, k)로 되돌리기

**명제 1 (전단사)**: 번호 매기기는 겹침도 빠짐도 없이 1:1입니다. 증명은
나눗셈: `k = 번호 ÷ 48²의 몫`, `r = 나머지 ÷ 48의 몫`, `c = 마지막 나머지`.
그래서 번호 하나를 뽑으면 자세가 **딱 하나로** 정해집니다.

## 물리 지지집합과 의미 지지집합 — 수식 (2) 풀이

정책이 9,216개 전부에서 뽑는 게 아닙니다. 두 단계 필터가 있습니다.

**1단계 — 물리 지지집합 A^phy**: 하드 유효(방 안·문 OK·안 겹침)인 번호만 남김.

**2단계 — 의미 지지집합**: 지금 놓는 가구가…
- 짝의 자식(협탁)이고 부모(침대)가 이미 놓였다면 → "짝 점수 q_e가 기준치
  이상인 자리들" C^pair를 만들고,
- 벽에 붙는 종류(옷장)라면 → "벽 점수가 기준치 이상인 자리들" C^wall을 만들어,
- 해당되는 집합들의 **교집합**을 취합니다.

**후퇴 규칙(fallback)**: 조건에 맞는 자리가 하나도 없으면 (교집합이 공집합),
그냥 물리 지지집합 전체로 돌아갑니다. 수식 (2)가 말하는 것이 이것입니다:

```
정책의 선택지 = 의미 후보가 있으면 의미 후보,
              없으면 물리 후보 전체
```

정직하게 짚을 두 가지:
- 이 필터는 **확률을 재분배할 뿐**, 새 자리를 만들거나 자리를 옮겨 주지 않습니다.
- 필터는 상태만 보고 결정되며(신경망 가중치와 무관), 그래서 학습 전후에 똑같이
  동작합니다.

## 전이와 종료 (수식 (3), 명제 2)

번호 a_t를 뽑으면 dec(a_t)의 좌표가 **그대로** 저장됩니다 — 반올림 보정,
스냅, 충돌 수리, 다시 뽑기, 가구 건너뛰기 전부 없음.

- 가구를 다 놓으면(t+1=N) → 완료
- 다음 가구를 놓을 물리 자리가 하나도 없으면 → 실패 (거기서 판 끝)
- 그 외 → 계속

**명제 2**: 모든 판은 최대 N수 안에 완료 또는 실패로 끝나고, 저장된 모든
자세는 놓일 당시 유효했다. 증명은 간단합니다 — 매 턴 가구가 정확히 1개씩
늘어나므로 N턴을 넘길 수 없고, 뽑는 자리가 항상 물리 지지집합 안에 있으므로
유효합니다.

## 보상과 목표 (수식 (4), (5))

자세한 점수는 3장에서 정의하고, 여기서는 뼈대만:

- **b(s_T)**: 판이 끝날 때 한 번 주는 최종 점수 (품질 + 통과 보너스 − 위반 벌점)
- **Φ(s_t)**: 중간 배치가 "얼마나 유망한지"를 재는 잠재함수 (저금통이라고
  생각하세요)
- **매 턴 보상** (수식 (4)):

```
r_t = (끝났으면 최종 점수) + γ×(다음 저금통) − (지금 저금통)
```

여기서 **1[…]** 는 지시함수입니다: 괄호 안이 참이면 1, 거짓이면 0. 그래서
최종 점수는 판이 끝나는 턴에만 들어옵니다.

- **학습 목표** (수식 (5)): 여러 방(장면)에 대해 평균 낸 "할인된 점수 총합"을
  최대로 만드는 정책 π*를 찾기. E는 평균(기댓값) 기호입니다.

**명제 3 (마르코프 충분성)**: 지금 상태 s_t와 행동 a_t만 알면 다음에 무슨 일이
일어날지 완전히 결정된다 — 과거 행동 이력은 추가 정보가 없다. 상태에 설계도,
놓인 가구 전부, 순서, 진행 상황이 다 들어 있기 때문입니다. (이게 성립해야
강화학습 이론이 적용됩니다.)
