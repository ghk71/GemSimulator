# 07. 부록 — 증명·알고리즘·평가 규칙 쉽게 풀기

## A. 증명 4개, 아주 쉽게

### 증명 1: 모든 점수는 0~1 (보조정리 1)

**전략**: "재료가 전부 0~1이고, 조립 방법이 0~1을 벗어나게 하지 않는다"를
차례로 확인.

- 재료: 과녁 커널(최대 1, 최소 0), 클립된 내적(잘라서 0~1), 넓이 비율(부분÷
  전체라 0~1), 램프 함수(정의상 0~1).
- 조립: 0~1끼리 곱하기 → 0~1. 0~1을 양수 거듭제곱 → 0~1. min/max → 0~1.
  Guard(최솟값^η × 기하평균^(1−η)) → 0~1.
- 빈 목록 규칙: 해당 없음 항목은 1(중립), 필요한 가구가 빠졌으면 0.
- 따라서 8개 항목, 가중 평균, 병목, 최종 Q, 잠재함수 Φ(≤κ) 전부 범위 안.

### 증명 2: 힌트 점수는 목표를 안 바꾼다 (정리 1)

매 턴 보상은 `b_t + γΦ(다음) − Φ(지금)`. 이걸 판 전체로 더하면 Φ들이
**도미노처럼 지워집니다**:

```
[b₀ + γΦ₁ − Φ₀] + γ[b₁ + γΦ₂ − Φ₁] + γ²[b₂ + γΦ₃ − Φ₂] + …
= (b들의 합) − Φ₀ + γ^T Φ_T
```

가운데 항들이 +γΦ₁ 다음 −γΦ₁ 식으로 짝지어 사라지는 것(망원 합)이 전부입니다.
남는 두 항도 0입니다: Φ₀는 빈 방이라 0, Φ_T는 종료 상태에서 0으로 **정의**했기
때문. 그래서 어떤 한 판에서든(평균이 아니라 매 판마다!) 힌트 포함 총점 = 원래
총점. 두 정책의 우열 비교가 절대 안 뒤집힙니다. 판 길이 T가 정책마다 달라도
성립합니다 — 소거를 "실현된 그 판"에서 했기 때문입니다.

### 증명 3: 가구 순번을 섞어도 결과 동일 (명제 4)

가구 목록에서 1번↔3번을 바꿔치기해도 (벽·문은 그대로):

1. 노드 변환은 모든 노드가 같은 신경망을 쓰므로, 출력도 같이 섞일 뿐 내용
   불변.
2. 어텐션에서 노드 i가 보는 "이웃 점수들의 집합"은 이름만 바뀐 같은 집합 →
   softmax 결과 동일 → 가중합도 동일 (더하기는 순서 무관).
3. 풀링은 전체 합이라 순서 무관, "현재 가구" 추출도 같은 의미의 노드를 가리킴.
4. 공간 텐서는 기하(어디에 뭐가 있나)로 만들지 저장 순번을 안 쓰므로 불변.
5. 따라서 FiLM, CNN, 로짓, 크리틱 값 전부 불변. (부동소수점 덧셈 순서에 따른
   미세 오차만 허용 범위로 선언.)

### 증명 4: 마스크드 확률과 정확한 실행 (보조정리 2 + 따름정리 1)

- 허용 자리가 1개 이상이고 점수가 유한하면, softmax 분모(허용 자리 exp들의
  합)는 양수·유한 → 나눗셈이 항상 성립 → 진짜 확률분포 (금지 자리 0, 허용
  자리 양수, 합 1).
- 뽑힌 정수 a는 명제 1(1:1 번호 매기기)에 의해 자세 하나로 확정되고, 전이
  규칙(수식 3)은 그 좌표를 **그대로** 저장 — 좌표를 만지는 다른 어떤 연산도
  전이 정의에 없음. 그래서 "확률을 계산한 그 행동"과 "실행된 그 자세"가
  동일합니다.
- 마스크는 상태만의 함수라 예전 정책과 지금 정책이 같은 지지집합을 공유 →
  PPO 비율(지금 확률÷예전 확률)의 분모가 0이 될 수 없음 → 비율이 항상 유한.

## B. 알고리즘 (수도코드를 말로)

```
매 학습 반복마다:
  여러 방(장면)을 뽑아 각각 한 판씩 플레이:
    빈 방에서 시작
    판이 안 끝났으면 반복:
      1. 그래프 + 지도 관측 만들기, 물리/의미 지지집합 계산
      2. 신경망 한 번 돌려 9,216개 점수 뽑기
      3. 마스크 씌우고 표준화해서 확률분포 만들기
      4. 자리 하나 샘플 → (상태, 행동, 로그확률, 크리틱값) 기록
      5. dec(자리번호)를 그대로 방에 놓기
      6. 다 놓았으면 완료 / 다음 가구 놓을 데 없으면 실패
      7. 힌트 보상 r = b + γΦ(다음) − Φ(지금) 기록
    GSAE로 어드밴티지 계산 (판 끝에서는 부트스트랩 0)
  어드밴티지 정규화
  미니배치로 PPO 업데이트 (같은 마스크·같은 변환을 재구성해서 비율 계산)
```

**평가 시에는**: 탐색용 온도 변환을 끄고, 마스크된 원시 점수의 argmax 한 판.
여러 개 뽑아 고르기·재시도·좌표 수정 일절 없음.

## C. 평가 지표의 정확한 규칙 (요약)

- **일반화 트랙의 Q**: 요청 가구가 다 있고 물리 위반이 없을 때만 품질 점수,
  아니면 0. Q.05는 장면별 점수의 하위 5% 지점. JPR은 "물리 + 기능 전부 통과"
  비율.
- **채점기 호출 수**: 후보를 채점기로 평가한 횟수만 셈. 신경망 순전파는 0회.
  시간(초)과는 다른 자원이라 따로 보고.
- **고정 트랙의 J와 J_net**: J는 놓인 가구 기준 → PSR과 함께 읽기. J_net은
  빠진 가구당 벌점. 두 트랙의 J와 Q는 **다른 목적함수라 절대 한 줄로 순위
  매기지 않음**.
- **좌표 전용 감사기**: 방 스펙 + 최종 좌표만 입력. 겹침·경계·문 스윙을
  직사각형 계산으로 재확인, 접근/앞면 넓이 비율, 짝 기하(간격·방향·헤드
  정렬), 4방향 연결 기반 보행 가능성(대각선 코너 끼어들기 방지), 벽 정렬
  분해(간격/겹침/코사인), 무게중심·분산 기반 구도 진단까지. 스타일·색·높이
  같은 미적 요소는 안 잼.
- **GVP**: 접근/짝/앞면/동선/문 5개의 위반율을 **점수로 합치지 않고** 각각
  보고.

## D. 비교 프로토콜의 해석 범위 (표로 선 긋기)

부록의 프로토콜 표가 하는 말: "이 행과 저 행을 함부로 비교하지 마라."

- OID-PPO vs CNN-only: 완전 매칭 비교 → 그래프 경로의 효과로 해석 가능.
- 대조군 3개: 진단용이지 외부 베이스라인이 아님.
- Myopic 탐색: 품질-계산량 곡선 정의용, 계산량 매칭 비교가 아님.
- "Project X": 알고리즘 계열의 자체 구현이지 공식 재현이 아님.
- "-style" / "transfer": 로컬 어댑터/이식이지 원 시스템 전체가 아니며, 전이
  행의 시간은 업스트림 생성 시간을 뺀 값임을 명시.
- Designer: 참조 1개, 분포 없음.

## E. 재현성

- 제출 패키지에는 코드 스냅샷(장면 생성/관측/정책/보상/학습/평가/집계),
  프로토콜·체크포인트 매니페스트, 그리고 **행동 단위 추적 기록**(정수 행동,
  (회전,행,열), 지지집합 소속, 로그확률, 디코드 자세, 저장 자세)이 들어감.
- 시험용 90개 방과 그 지표는 학습·체크포인트 선택에 일절 사용 안 함.
- 평가는 "이상하면 멈춤"(fail-closed): 매니페스트·지지집합·행동번호·
  디코드-저장 자세가 하나라도 안 맞으면 결과를 내지 않고 실패 처리.
- 고정 트랙은 출처가 섞여 있어서(일부 API 호출은 재실행 불가) 좌표 재검증까지만
  보장 — 이 경계를 라벨("local", "-style", "transfer")로 표시.
